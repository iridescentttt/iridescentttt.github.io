机器人PRM路径规划

No tag
Publish Date:   2021-12-08

使用 PRM 在 webots 上实现了 PRM (概率路线图)路径规划, 让小车能够从起点自动导航走到终点.

给定一个地图, 我们怎么根据地图来应用 PRM 算法构建路径呢?

PRM 算法

PRM(Probabilistic Roadmaps) 是一种基于图搜索的方法. 主要分成以下两个部分.

  • 使用随机采样点来构造地图.
  • 在采样的点上跑 dijsktra

好的我们已经学会 PRM 了. 你已经可以达到以下效果了.

构建地图.

路径规划:

开始跑.

如果没学会的话就看看下面的报告吧.

1. 安装 opencv 库

本次实验使用了 c++ 版本中的 opencv 库来进行图片的读取, 所以首先得安装 opencv 的环境.

根据 webots 的官方教程(https://www.cyberbotics.com/doc/guide/using-webots-makefiles)中的解析, 可以直接通过在 controller 文件夹下的 makefile 文件中指定链接库, 只要指定链接库的位置和相应链接文件即可.

所以考虑尝试从 opencv 的源码进行编译安装然后使用 makefile 链接这些文件. 但是在花费较长时间编译完成生成相应的 dll 文件之后, 使用 webots 中的 gcc 编译器却无法进行链接. 经过一番查找原因后, 发现 webots 中的 gcc 编译器是 32 位的, 而本地 mingw 编译器是 64位的, 因而后者编译生成的文件前者无法进行链接. 无奈重新下载了别人编译好的 opencv 库(https://gitee.com/kirigaya/opencv_built_by_gcc_on_-windows)进行链接, 总算能使用了.

最后 makefile 中加入下列语句即可链接下载好的 opencv 库.

### 更改为opencv库对应位置
INCLUDE = -I"C:\Users\ADMINA\Desktop\opencv\build\include"
LIBRARIES = -L"C:\Users\ADMINA\Desktop\opencv\build\lib" -lopencv_core452 -lopencv_imgcodecs452 -lopencv_imgproc452 -lopencv_calib3d452 -lopencv_dnn452 -lopencv_features2d452 -lopencv_flann452 -lopencv_gapi452 -lopencv_highgui452 -lopencv_ml452 -lopencv_objdetect452 -lopencv_photo452 -lopencv_stitching452 -lopencv_video452 -lopencv_videoio452
2. 坐标系说明

在原来的 $800\times 600$ 的地图中, 本次实验使用一个坐标 $(x,y)$ 来进行表示每一个像素点. 坐标系中, 地图左上角为原点, x 轴的正方向向下, y 轴的正方向向右. 另外, 为了方便存储, 每一个坐标 $(x,y)$ 对应有一个一维的坐标用 pixel 表示.
$$
pixel=x\times 600+y
$$

3. GPS 使用

在本次实验中, 需要进行确定小车位置和车头朝向. 这个过程通过两个 GPS 来进行实现.

  • 在车身有一个 GPS, 用来确定小车位置 $(cur_x,cur_y)$.
  • 在车头有一个 GPS, 用来确定车头位置 $(head_x,head_y)$

由于 GPS 不好进行原图上的定位, 还需要将 GPS 坐标转化为原地图上的坐标. 经过观察可以发现, 原地图中点处的 GPS 坐标为 (0,0), 而且边界处 GPS 的数值完全与地图的大小吻合. 所以只要经过一定的线性变化就可以将 GPS 位置转化为地图坐标.

综上, 我们有下列辅助函数.

// 根据行列计算对应像素值
void getPixel(int x, int y, int &pixel)
{
    pixel = x * cols + y;
}

// 根据pixel计算对应的行和列
void getPosFromPixel(int &x, int &y, int pixel)
{
    x = pixel / cols;
    y = pixel % cols;
}

// 将GPS位置转化为地图中坐标
void getPosFromGps(int &x, int &y, double gps_x, double gps_y)
{
    x = (-gps_x + total_x / 2) / gps_dx;
    y = (gps_y + total_y / 2) / gps_dy;
}

其中 total_x 和 total_y 为地图总大小, gps_dx 和 gps_dy 为一个像素所占的 gps 数值多少.

4. 读入迷宫

使用 opencv 中的 imread 的 api 进行读入迷宫的操作, 结果存在一个 Mat 对象中.

之后我们将其中每一个像素的 RGB 值按照一维形式的下标存于对应的数组中, 方便检查. 同时求取 gps_dx 和 gps_dy, 它们表示一个像素在两个坐标轴方向所占的 gps 数值多少(即刻度尺大小).

Mat maze;
int rows, cols, max_pixel;
int maze_r[1000000], maze_g[1000000], maze_b[1000000];
// 读入迷宫
void readMaze()
{
    maze = imread("../../maze.png");
    rows = maze.rows, cols = maze.cols;
    int pixel;
    max_pixel = rows * cols;
    for (int x = 0; x < rows; ++x)
    {
        for (int y = 0; y < cols; ++y)
        {
            getPixel(x, y, pixel);
            maze_r[pixel] = (int)maze.at<Vec3b>(x, y)[0], maze_g[pixel] = (int)maze.at<Vec3b>(x, y)[1], maze_b[pixel] = (int)maze.at<Vec3b>(x, y)[2];
        }
    }
    gps_dx = total_x / rows;
    gps_dy = total_y / cols;
}
5. 采样点并构建邻接矩阵

读入地图之后, 就可以通过随机采样的方式在原图上进行采样点.

由于使用了一维形式的坐标, 进行采样就特别方便, 只需要在 $[0,max_pixel]$ 上进行均匀采样即可.

采样的时候, 需要判断采样的点是否在障碍上. 地图上的障碍是黑色的, RGB 数值为 $(0,0,0)$. 为了增加准确性, 我们把 RGB 三个通道的数值同时小于 100 的都认为是障碍. 另外, 考虑到小车本身具有大小, 为了防止采样的点过于接近障碍, 所以一个 $20\times 20$ 邻域内的存在障碍的点我们也不予采样.

最终的判断函数如下所示.

// 判断一个点是否在障碍上
int judge(int pixel)
{
    int x, y;
    getPosFromPixel(x, y, pixel);
    for (int i = -10; i <= 10; i++)
    {
        for (int j = -10; j <= 10; j++)
        {
            if (x + i < 0 || x + i >= rows)
                continue;
            if (y + j < 0 || y + j >= cols)
                continue;
            getPixel(x + i, y + j, pixel);
            if (maze_r[pixel] <= 100 && maze_g[pixel] <= 100 && maze_b[pixel] <= 100)
                return 0;
        }
    }
    return 1;
}

判断完一个点能否被采样之后, 接着我们来构造邻接矩阵.

满足下列条件的两个点将被连接起来.

  • 两点之间的距离小于一个设定好的上限 $distance$. 本次实验中计算距离利用的是地图上的坐标, 同时设置 $distance=100$.

  • 两点之间的路径不经过障碍. 这个通过一个 checkPath 函数来进行实现, 具体就是构建两点之间的直线, 然后在该直线上采样点. 然后利用上面的判断函数来判断采样到的这些点是否在障碍上, 如果有一个点在障碍上, 那么这条路径就是不合法的. 本次实验中设置采样点的个数为 10.

    // 判断道路是否合法
int checkPath(int x1, int y1, int x2, int y2, int split)
{
    double dx = (x2 - x1) / split, dy = (y2 - y1) / split, x, y;
    int pixel;
    for (int i = 0; i < split; i++)
    {
        x = x1 + dx * i;
        y = y1 + dy * i;
        getPixel((int)x, (int)y, pixel);
        if (!judge(pixel))
            return 0;
    }
    return 1;
}

采样的点由 node 这个 vector 存起来, 连接的点通过红线连接起来, 然后整张图片写进图片 road.png 中. 最终的采样函数如下所示.

vector<int> node;
void sampleNode(int node_num, int split)
{
    random_device rd;
    mt19937 gen(rd());
    uniform_int_distribution<> dis(0, max_pixel);
    unordered_map<int, int> vis;
    int pixel;
    double distance = 100;
    while ((int)node.size() < node_num)
    {
        pixel = dis(gen);
        if (vis[pixel])
            continue;
        vis[pixel]++;
        if (judge(pixel))
            node.push_back(pixel);
    }
    sort(node.begin(), node.end());
    for (int i = 0; i < node_num; i++)
    {
        getPosFromPixel(nodex[i], nodey[i], node[i]);
    }
    for (int i = 0; i < node_num; ++i)
    {
        for (int j = i + 1; j < node_num; ++j)
        {
            if (calDis(nodex[i], nodey[i], nodex[j], nodey[j]) < distance && checkPath(nodex[i], nodey[i], nodex[j], nodey[j], split))
            {
                g[i].push_back(j);
                g[j].push_back(i);
                line(maze, Point(nodey[i], nodex[i]), Point(nodey[j], nodex[j]), Scalar(0, 0, 255), 1);
            }
        }
    }
    imwrite("road.png", maze);
}
6. Dijsktra 算法与最终实现

已知采样点和它们之间的连通性之后, 我们通过 dijsktra 算法得到最短路.

需要注意的是, 我们需要首先把起点和终点放进采样点中, 整条路径从起点开始, 到达终点结束. 通过对地图的观察, 我们可以得到起点坐标为 (44,547), 终点坐标为 (756,35).

dijsktra 算法的整个流程如下.

  • 从优先队列中选取一个距离最近的没有到达过的节点 u.
  • 寻找节点 u 的所有的没有到达过的邻居, 更新到达这些邻居的距离, 并把更新过的节点加入到优先队列中. 这个过程中, 记录更新的节点所在路径上一个节点即为 u.
  • 重复上述步骤直至到达终点, 然后回溯便能够得到整条路径.

实现 dijsktra 算法需要实现一个自定义的结构, 可以供优先队列按照距离排序.

struct myNode
{
    int u;
    double d;
    friend bool operator<(const myNode &a, const myNode &b)
    {
        return a.d > b.d;
    }
};

然后按照上述步骤实现 dijsktra 算法得到最终路径, 并使用绿色线段标出来写入到图片 path.png 中. 需要注意进行反向才能够得到从起点到达终点顺序的路径.

最后实现函数如下所示.

void work()
{
    int node_num = 600, pixel, split = 10, start_node = 0, end_node = 0;
    priority_queue<myNode> q;
    unordered_map<int, int> vis;

    pair<int, int> start = make_pair(44, 547);
    pair<int, int> end = make_pair(756, 35);

    getPixel(start.first, start.second, pixel);
    node.push_back(pixel);
    getPixel(end.first, end.second, pixel);
    node.push_back(pixel);

    sampleNode(node_num, split);

    for (int i = 0; i < (int)node.size(); i++)
    {
        if (nodex[i] == start.first && nodey[i] == start.second)
        {
            start_node = i;
            dis[start_node] = 0;
        }
        else if (nodex[i] == end.first && nodey[i] == end.second)
        {
            end_node = i;
            dis[i] = 1e6;
        }
        else
        {
            dis[i] = 1e6;
        }
    }
    q.push(myNode{start_node, 0});
    myNode tmp;
    while (q.size() > 0)
    {
        tmp = q.top(), q.pop();
        int u = tmp.u;
        if (vis[u])
            continue;
        vis[u]++;
        if (u == end_node)
        {
            break;
        }
        for (int i = 0; i < (int)g[u].size(); i++)
        {
            int v = g[u][i];
            double ddis = calDis(nodex[u], nodey[u], nodex[v], nodey[v]);
            if (dis[v] > dis[u] + ddis)
            {
                fa[v] = u;
                dis[v] = dis[u] + ddis;
                q.push(myNode{v, dis[v]});
            }
        }
    }

    for (int i = end_node; i != start_node; i = fa[i])
    {
        path.push_back(i);
        line(maze, Point(nodey[i], nodex[i]), Point(nodey[fa[i]], nodex[fa[i]]), Scalar(0, 255, 0), 10);
    }
    imwrite("path.png", maze);
    path.push_back(start_node);
    reverse(path.begin(), path.end());
}
7. 小车导航

得到一条可行的路径之后, 需要解决的问题就是让小车按照得到的路径行驶到终点. 一个简单的想法就是按照之前的作业直接替换图片然后进行小车巡线. 但是这种方案显然不够稳定并且替换掉了地图.

所以我们基于之前的车头和车身 GPS 构造了一种基于向量的行驶方案. 我们可以先由车头和车身的 GPS 进行相应的线性变换得到对应的坐标 $(head_x,head_y)$ 和 $(cur_x,cur_y)$. 同时我们也可以确定下一个点的位置 $(target_x,target_y)$

整个导航过程如下所示.

  • 确定路径上下一个目标点的位置 $(target_x,target_y)$. 如果已经到达终点, 则停止电机并跳出循环.
  • 通过 GPS 确定车头位置$(head_x,head_y)$ 和车身位置 $(cur_x,cur_y)$.
  • 计算得到两个向量 $H=(head_x-cur_x,head_y-cur_y)$ 和 $T=(target_x-cur_x,target_y-cur_y)$. 它们分别表示车头朝向的向量以及小车到达目标点的向量.
  • 计算向量 $H$ 和 $T$ 的夹角, 这个可以通过计算点乘然后计算得到夹角的余弦值间接表示出来.
    • 如果夹角小于 5 度, 我们就认为小车前进方向正确, 接下来的动作就是直行.
    • 如果夹角大于 5 度, 这个时候我们就需要进行判断小车需要进行左转还是右转. 这个过程可以通过计算 $H$ 和 $T$ 的叉乘 $H \times T$得到. 如果结果大于0, 就说明 $T$ 在 $H$ 逆时针方向, 小车应该向左转. 如果结果小于0, 就说明 $T$ 在 $H$ 顺时针方向, 小车应该向右转.

通过向量点乘和叉乘导航的具体实现如下所示.

// 到达终点停止电机
if (cur >= (int)path.size())
{
    for (int i = 0; i < 4; ++i)
    {
        speed[i] = 0;
    }
    for (int i = 0; i < 4; ++i)
    {
        motors[i]->setVelocity(speed[i]);
    }
    continue;
}
int target = path[cur];
double gps_x = gps->getValues()[1], gps_y = gps->getValues()[0];
double head_gps_x = head->getValues()[1], head_gps_y = head->getValues()[0];
// 得到具体坐标
getPosFromGps(cur_x, cur_y, gps_x, gps_y);
getPosFromGps(head_x, head_y, head_gps_x, head_gps_y);
fflush(stdout);
if (calDis(cur_x, cur_y, nodex[target], nodey[target]) < 5)
{
    cur++;
    continue;
}
int keyValue = 0;
cout << "pos: " << cur_x << " " << cur_y << endl;
cout << "head: " << head_x << " " << head_y << endl;
cout << "target: " << nodex[target] << " " << nodey[target] << endl;
// 通过计算点乘计算余弦值
double dot = (head_x - cur_x) * (nodex[target] - cur_x) + (head_y - cur_y) * (nodey[target] - cur_y);
double norm = calDis(0, 0, head_x - cur_x, head_y - cur_y) * calDis(0, 0, nodex[target] - cur_x, nodey[target] - cur_y);
double res = dot / norm;
// 计算叉乘
double cross = (head_x - cur_x) * (nodey[target] - cur_y) - (head_y - cur_y) * (nodex[target] - cur_x);
if (abs(res) >= 0.995)// 直走
{
    keyValue = 'W';
}
else if (cross > 0)// 左转
{
    keyValue = 'A';
}
else if (cross < 0)// 右转
{
    keyValue = 'D';
}
